# task.md Reference Format

This file documents the structure the planner agent must follow when
creating `task.md`. The planner creates this file from scratch on the
first iteration — it is NOT auto-generated by the orchestrator.

---

## Expected Sections

### 1. Metadata
```markdown
## Metadata
- **jira_ticket**: <ticket ID>
- **description**: <JIRA description>
- **project_path**: <absolute path to the external Spring Boot project>
- **current_phase**: PLANNING | IMPLEMENTING | REVIEWING | TESTING | COMPLETE | NEEDS_USER_HELP
- **iteration**: <current iteration number>
- **max_iterations**: <max allowed>
- **overall_status**: IN_PROGRESS | COMPLETE | FAILED
```

### 2. Tasks
```markdown
## Tasks

### Task N: <Short Title>
- **status**: PLANNED | IMPLEMENTED | APPROVED | REWORK_NEEDED | DONE
- **file**: <relative path to implementation file>
- **test_file**: <relative path to unit test file>
- **description**: <detailed implementation instructions>
- **review_feedback**: <filled by reviewer agent>
- **rework_notes**: <filled by planner on rework iterations>
```

### 3. Test Results
```markdown
## Test Results
- **last_run**: <timestamp or N/A>
- **status**: NOT_RUN | PASSING | FAILING
- **total**: <number>
- **passed**: <number>
- **failed**: <number>
- **errors**: <number>
- **output_summary**: <brief Maven output>
- **failure_details**: <specific failure messages>
```

### 4. Iteration History
```markdown
## Iteration History

### Iteration N
- **planner**: <what was planned/re-planned>
- **implementer**: <what was implemented>
- **reviewer**: <what was approved/rejected>
- **tester**: <test results summary>
```

## Status Flow

```
PLANNED → IMPLEMENTED → APPROVED → DONE
                      ↘ REWORK_NEEDED → PLANNED (next iteration)
```
